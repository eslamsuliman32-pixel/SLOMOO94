# سجل القرارات المحسومة

قرارات لا يعاد فتحها إلا بقرار صريح من المالك. أي قرار جديد يُضاف هنا بصف مرقم.

| # | القرار | التفصيل |
|---|---|---|
| D1 | الإنسان هو الصانع | لا ميزة تكتب الراب بدلًا عن المالك؛ الآلة تكشف وتعرض وتقيس فقط |
| D2 | استخدام شخصي غير تجاري | مستخدم واحد — يبرر البنية صفرية التكلفة وغياب أنظمة تعدد المستخدمين |
| D3 | بيئة نظيفة + استيراد انتقائي | القديم مرجع منطقي للقراءة؛ لا نسخ-لصق كود إطلاقًا (بروتوكول SYSTEM_MAP) |
| D4 | البنية صفرية التكلفة | GitHub/Codespaces/Vercel/Firestore/R2/Kaggle — التكلفة الشهرية 0$ |
| D5 | المعالجة في المعمل السحابي، والمتصفح للعرض والتشغيل فقط | أعلى دقة تحليلية دون قيود المتصفح، وبلا فقد جودة في التشغيل |
| D6 | العزل ثلاثي المسارات افتراضيًا | درامز/باص/ميلودي؛ مسار الفوكال يتفعل تلقائيًا فقط عند رفع أغنية مرجعية |
| D7 | الدمج المعرفي للوزن | علم العروض الفراهيدي طبقة معرفية، ولغة التشغيل ●/▬ (بصمة الوزن) بدل حفظ التفعيلات |
| D8 | مرونة القافية بثلاثة أوضاع | أ: عائلات المخارج · ب: دلالات الحروف (مصدر: أسرار الحروف) · ج: خريطة قوافي الأقسام (رباعيات/سداسيات) |
| D9 | التمثيلات بنظام المزيج | أرشيف المالك أولًا (تصنيف/بحث/ربط)؛ الاقتراح التوليدي بطلب صريح فقط |
| D10 | طبقة LLM قابلة للتبديل | Gemini المجاني افتراضيًا؛ Claude API ترقية اختيارية (رصيد مسبق الدفع، منفصل عن اشتراك claude.ai) |
| D11 | عيوب القافية إشارات محايدة | المحرك يعرض العيب كمعلومة ولا يصحح ولا يحكم |
| D12 | التنفيذ بالدُفعات الواسعة | كل دورة عمل تنجز مجموعة خطوات مترابطة دفعة واحدة، والدفع للمستودع مرة واحدة في نهايتها |
