# Beat Profile JSON — النسيج الرابط بين التحليل والإبداع

## الفكرة
محلل البيت الحقيقي (`maqam_beat_analyzer.html`) يحلل الصوت مرة واحدة فقط، ويُصدّر
النتيجة كـ **Beat Profile JSON** — ملف واحد يحمل كل ما يحتاجه أي مستهلك لاحق
(Command Center، Puzzle Grid، Training) ليعرض نفس البيانات **دون إعادة أي تحليل
طيفي**. هذا يطابق قرار المشروع D5: التحليل الثقيل مرة، والعرض خفيف بعدها دائمًا.

## الملفات المسلَّمة في هذه الدفعة
| الملف | الدور |
|---|---|
| `maqam_beat_analyzer.html` | محدَّث: يصدّر ويستورد Beat Profile فعليًا (أزرار ⬇⬆) |
| `beat_profile.schema.json` | مخطط JSON Schema رسمي (Draft-07) موثِّق لكل حقل |
| `maqam_beat_profile_pipeline.png` | ملصق مرئي لخط الأنابيب الكامل |
| `beat_profile_integration.md` | هذه الوثيقة |

## دورة الحياة
```
ملف صوتي (mp3/wav)
   → محلل البيت الحقيقي (STFT/BPM/كيك-سنير-808/أقسام)
   → زر "تصدير Beat Profile JSON"
   → ملف .json واحد يمتثل للمخطط الرسمي
   → أي مستهلك (Command Center, Puzzle Grid, Training) يستورده
     ويُعيد بناء نفس الشبكة والأقسام والبارات فورًا، بلا صوت حتى
```

## أهم الحقول (التفصيل الكامل في `beat_profile.schema.json`)
- **`tempo.bpm` / `confidencePercent`** — الإيقاع المكتشف فعليًا وموثوقيته
- **`sections[]`** — أقسام البيت الحقيقية (من/إلى بار، مستوى الطاقة، الوسم)
- **`bars[].slotTimesSec`** — **الأهم**: الزمن الحقيقي بالثانية لكل خانة من الـ16،
  محسوب من النبضات **المُنقَّحة** على الصوت الفعلي لا من معادلة BPM جامدة.
  هذا الحقل وحده يكفي لأي مشغّل صوتي خارجي ليُزامن ميترونومه دون أي حساب إضافي.
- **`bars[].slots[].{kick,snare,e808,rawOnset}`** — قيم منطقية صِرفة، حقيقية 100%
- **`lyrics`** — طبقة اختيارية: مقاطع مُقطَّعة عروضيًا + نسبة تزامن النبر الفعلية

## التحقق الذي تم إجراؤه
1. ✅ التصدير الفعلي من تحليل حقيقي يمتثل بالكامل لـ `beat_profile.schema.json`
   (تحقق آلي عبر مكتبة `jsonschema`، Draft-07)
2. ✅ اختبار **جولة كاملة (round-trip)**: نفس الملف الصوتي → تحليل حي → استخراج
   الشبكة لكل الـ8 بارات ← مقارنة حرفية بشبكة مُعاد بناؤها من JSON فقط (بلا صوت
   إطلاقًا) → **تطابق تام صفر اختلافات** عبر كل البارات
3. ✅ الاستيراد النقي (JSON بلا ملف صوتي مرفق) يعرض الشبكة بصريًا بشكل صحيح،
   ويمنع محاولة التشغيل الصوتي برسالة واضحة بدل الانهيار
4. ✅ حالة حافة صفرية أُصلحت أثناء الاختبار: بار البداية كان يحمل زمنًا سالبًا
   طفيفًا (−40ms) ناتجًا عن تمديد الشبكة للخلف؛ تم تثبيته على صفر لضمان توافق
   المخطط ومنع فشل أي `AudioBufferSourceNode.start()` مستقبلي

## للمطوّرين — الدمج في أي واجهة جديدة
انسخ نمط الدالتين التاليتين من `maqam_beat_analyzer.html` مباشرة:
- `buildBeatProfile()` — تجميع التحليل الحالي إلى الشكل الموحّد
- `hydrateFromProfile(profile)` — إعادة بناء `S.analysis` و `S.text.bars` من أي
  ملف JSON متوافق، دون أي استدعاء لـ `analyze()`

هذا هو العقد الكامل بين الطبقتين — أي واجهة تستورد نفس الحقول تحصل على نفس
الدقة والمزامنة تلقائيًا.
